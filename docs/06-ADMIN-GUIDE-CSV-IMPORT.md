## 🎯 Overview

This guide explains the complete workflow for importing users and teams via CSV files, including automatic Auth account creation for proper authentication.

Before importing any data:
1. ✅ Firebase project configured and running

## 📝 Step-by-Step Process

### Step 1: Prepare User CSV File

**Format:**
```csv
name,email,role,uid
```

**Example:**
```csv
name,email,role,uid
John Doe,s1@gehu.ac.in,student,
Jane Smith,s2@gehu.ac.in,student,
Dr. Bob Johnson,f1@gehu.ac.in,faculty,
Prof. Alice Brown,f2@gehu.ac.in,faculty,
```

**Important Notes:**
- Leave `uid` column empty for CSV imports
- UIDs will be automatically generated by Firebase Auth
- Email must end with `@gehu.ac.in`
- Role must be: `student`, `faculty`, `external`, or `admin`

---

### Step 2: Import Users (Manage Users Page)

1. **Navigate:** Admin Dashboard → Manage Users
2. **Click:** "Import CSV" button
3. **Paste:** CSV data into textarea
4. **Click:** "Import" button

**What Happens:**
```
Import Process:
┌─────────────────────────────────────────┐
│ 1. Validate CSV format                  │
│    → Check headers (name, email, role)  │
│    → Validate email domains             │
│    → Check for duplicates               │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ 2. Create Firestore User Documents      │
│    → Document ID = email                │
│    → Fields: name, email, role          │
│    → hasAuthAccount = false initially   │
│    → sessionId = active session         │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ 3. Auto Create Auth Accounts            │
│    → AuthSyncService.bulkCreateAuth()  │
│    → Password = Gehu@2025 (default)     │
│    → Get Auth UID from Firebase         │
│    → Sync UID to Firestore             │
│    → Set hasAuthAccount = true          │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│ 4. Update UI                            │
│    → Table shows "Active" badges        │
│    → Stats updated                       │
│    → Success toast displayed            │
└─────────────────────────────────────────┘
```

**Expected Output:**
- ✅ Green success toast: "X user(s) imported successfully"
- ✅ Second toast: "Auth accounts created for X user(s)"
- ✅ Users table shows all users with "Active" Auth status

**Troubleshooting:**
- ❌ If Auth creation fails: Click "Create Auth Accounts (X)" button to retry
- ❌ If validation fails: Check console for detailed error messages
- ❌ If duplicates found: Remove duplicate emails from CSV

---

### Step 3: Verify Auth Accounts Created

**Check Auth Status Column:**
- 🟢 **Active** = User has Auth account (can login)
- 🟠 **Pending** = User needs Auth account (cannot login)

**If Some Users Show "Pending":**
1. Check "Create Auth Accounts" button shows correct count
2. Click button to create remaining accounts
3. Watch progress dialog
4. Verify all users show "Active" after completion

---

### Step 4: Prepare Team CSV File

**Format:**
```csv
teamName,projectName,mentorId,leaderId,memberIds
```

**Example:**
```csv
teamName,projectName,mentorId,leaderId,memberIds
AI Research Team,AI Chatbot System,f1@gehu.ac.in,s1@gehu.ac.in,s2@gehu.ac.in;s3@gehu.ac.in;s4@gehu.ac.in
Web Dev Team,E-Commerce Platform,f2@gehu.ac.in,s5@gehu.ac.in,s6@gehu.ac.in;s7@gehu.ac.in
Mobile App Team,Health Tracker,,s8@gehu.ac.in,s9@gehu.ac.in;s10@gehu.ac.in
```

**Field Descriptions:**
- `teamName` (required): Team's display name
- `projectName` (optional): Project title
- `mentorId` (optional): Faculty email (can be empty)
- `leaderId` (required): Team leader's email
- `memberIds` (optional): Additional member emails (semicolon or comma separated, max 3)

**Important Notes:**
- Max 4 members per team (leader + 3 additional)
- All emails must be from imported users
- All users must have Auth accounts (check "Manage Users")
- Leader will be included in members array automatically

### Step 5: Import Teams (Manage Teams Page)

1. **Navigate:** Admin Dashboard → Manage Teams
2. **Click:** "Import CSV" button
3. **Read:** Prerequisites warning carefully
4. **Paste:** CSV data into textarea
5. **Click:** "Import" button

**What Happens:**
```
Team Import Process:
┌──────────────────────────────────────────┐
│ 1. Validate CSV Format                   │
│    → Check required headers              │
│    → Validate email formats              │
│    → Check for duplicate students        │
└────────────────┬─────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────┐
│ 2. Validate Students Exist               │
│    → Check Firestore users/{email}       │
│    → Verify role = student               │
│    → CHECK AUTH ACCOUNT EXISTS ⚡        │
│    → Get real Auth UID                   │
└────────────────┬─────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────┐
│ 3. Validate Mentors Exist (if provided) │
│    → Check Firestore users/{email}       │
│    → Verify role = faculty               │
│    → CHECK AUTH ACCOUNT EXISTS ⚡        │
│    → Get real Auth UID                   │
└────────────────┬─────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────┐
│ 4. Generate Project Numbers              │
│    → Use transaction for atomicity       │
│    → Counter: counters/project_{session} │
│    → Generate team code (T001, T002...)  │
└────────────────┬─────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────┐
│ 5. Create Team Documents                 │
│    → teams/{team_session_number}         │
│    → leaderId = Auth UID (not email) ⚡  │
│    → mentorId = Auth UID (if assigned)   │
│    → members = [emails] (array)          │
│    → sessionId = active session          │
└────────────────┬─────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────┐
│ 6. Update User Documents                 │
│    → users/{email}.teamId = teamId       │
│    → users/{email}.teamCode = teamCode   │
│    → users/{email}.projectNumber = num   │
└─────────────────────────────────────────┘
```

**Expected Output:**
- ✅ Green success toast: "X team(s) imported successfully"
- ✅ Teams table shows all teams with correct details
- ✅ Students can see their team in dashboard

**Common Errors:**

❌ **"Student X doesn't have an Auth account"**
- **Cause:** User imported but Auth account not created
- **Solution:** Go to "Manage Users" → Click "Create Auth Accounts (X)"
- **Then:** Retry team import

❌ **"Mentor X doesn't have an Auth account"**
- **Cause:** Faculty imported but Auth account not created
- **Solution:** Go to "Manage Users" → Click "Create Auth Accounts (X)"
- **Then:** Retry team import

❌ **"Student X appears in multiple teams"**
- **Cause:** Same student email in multiple teams
- **Solution:** Remove duplicate assignments from CSV

❌ **"User X not found in users collection"**
- **Cause:** User not imported yet
- **Solution:** Import users first in "Manage Users"

## 🔐 Default Login Credentials

After CSV import, all users can login with:
- **Email:** Their imported email (e.g., `s1@gehu.ac.in`)
- **Password:** `Gehu@2025` (default for all CSV imports)

### 🔔 Password Change Prompt

**On first login with default password:**
1. User enters email + `Gehu@2025`
2. Login succeeds
3. **Password Change Dialog appears automatically**
4. User sees security warning about shared default password
5. User can either:
   - **Change Password Now** (recommended) → Set new password → Access dashboard
   - **Skip for Now** (with warning) → Access dashboard with default password

**After password is changed:**
- Dialog will NOT appear on subsequent logins
- User must remember their new password
- Password change tracked in user profile

### ⚠️ IMPORTANT SECURITY NOTICE

- All CSV-imported users share the same initial password
- **Strong recommendation:** Change password immediately on first login
- Future logins will require the new password (or default if skipped)
- Users can change password later from profile settings (coming soon)
- Admins should communicate default password securely

**Security Features:**
- ✅ Automatic password change prompt on first login
- ✅ Clear warning about shared default password
- ✅ Validation: minimum 6 characters, matching confirmation
- ✅ Status tracking: knows if user changed password
- ⚠️ Skip option available (user choice with warning)

## 📊 Technical Details

### Auth UID vs Firestore UID

**OLD (Incorrect) Pattern:**
```javascript
// ❌ Used email as UID - security risk
const studentData = {
  uid: email, // WRONG
  email: email
};
```

**NEW (Correct) Pattern:**
```javascript
// ✅ Uses real Auth UID from Firebase
const authCheck = await AuthSyncService.checkUserExistsInFirestore(email);
const studentData = {
  uid: authCheck.uid, // Real Auth UID
  email: email,
  hasAuthAccount: true
};
```

### Email as Common Identifier

```
Auth Collection           Firestore Users Collection
┌──────────────┐         ┌────────────────────────┐
│ UID: abc123  │◄───────►│ Document ID: email     │
│ Email: s1@.. │  Link   │ uid: abc123            │
│ Password: ** │         │ hasAuthAccount: true   │
└──────────────┘         └────────────────────────┘
```

**Why Email as Document ID?**
- Enables efficient Firestore security rules
- Simplifies role-based access control
- Consistent identifier across systems
- Easy to query by email

**Why Store UID in Firestore?**
- Team leadership requires Auth UID
- Mentor assignment requires Auth UID
- Enables proper user references
- Maintains Auth as source of truth

## 🚨 Error Recovery

### Scenario 1: Auth Creation Failed During User Import

**Symptoms:**
- Users imported to Firestore
- Some/all show "Pending" Auth status
- "Create Auth Accounts" button visible

**Recovery:**
1. Click "Create Auth Accounts (X)" button
2. Wait for bulk creation to complete
3. Verify all users show "Active"
4. Proceed with team import

### Scenario 2: Team Import Failed (Missing Auth Accounts)

**Symptoms:**
- Error toast: "X student(s) don't have Auth accounts"
- Teams not created
- Console shows detailed errors

**Recovery:**
1. Go to "Manage Users" page
2. Create missing Auth accounts
3. Return to "Manage Teams" page
4. Retry team CSV import

### Scenario 3: Partial Team Import Success

**Symptoms:**
- Some teams created successfully
- Some teams failed with errors
- Mixed success/failure toast

**Recovery:**
1. Check console for specific errors
2. Fix issues in CSV (remove successful teams)
3. Retry import with only failed teams

## 🎓 Best Practices

1. **Import Order:** Always users before teams
2. **Verify Auth:** Check Auth status before team import
3. **Test Small:** Import 5-10 users first to test workflow
4. **Backup Data:** Export existing data before bulk import
5. **Communicate:** Share default password securely with users
6. **Change Passwords:** Encourage immediate password changes
7. **Validate CSV:** Check format carefully before import
8. **Check Logs:** Review console for any warnings/errors

## 🔄 Update Process (For Existing Deployments)

If you have users imported before Auth sync was implemented:

1. **Identify Old Users:**
   - Check for users with `uid === email`
   - Look for users with `hasAuthAccount: false`

2. **Create Auth Accounts:**
   - Use "Create Auth Accounts" button in Manage Users
   - This will create Auth accounts and update UIDs

3. **Update Team References:**
   - Teams with old email-based UIDs need updating
   - Future migration script will automate this

4. **Test Login:**
   - Verify users can login with Gehu@2025
   - Check dashboard access works correctly
